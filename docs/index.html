HOW IT WORKS
Connect. Operate. Audit.
A thin operations console — not a cluster installer or a GitOps engine. Point it at your cluster and registry and run it.
STEP 01
Connect
Point inInfra at your cluster via a ServiceAccount in-cluster or your kube context locally. RBAC is scoped to your MANAGED_NAMESPACES allowlist.
STEP 02
Operate
Scale and restart workloads, edit env, stream logs, browse PVC files, trigger Jenkins builds, and roll back deploys — all from the browser.
STEP 03
Audit
Every mutation is recorded to Postgres and attributed to the acting user, exposed as a cursor-paginated audit log.

ARCHITECTURE
A Rust API and a Next.js web, behind one ingress.
The browser calls same-origin /api/*, proxied to the Rust API. The API talks to the Kube API via kube-rs, persists audit + config to Postgres, and optionally integrates Jenkins and ECR.
SECURITY
Secure by default, least privilege by design.
Hashed credentials, stateless sessions, a namespace allowlist, role enforcement, and tightly scoped RBAC. Always serve over TLS.
argon2id password hashing
Admin and user passwords are hashed with argon2id — plaintext is never stored or logged.
HS256 HttpOnly session cookie
Login mints a stateless HS256 JWT in an HttpOnly, Secure, SameSite=Lax cookie with a 12h TTL.
Namespace allowlist
The API only reads and patches the configured MANAGED_NAMESPACES; mutating handlers validate the target.
Role enforcement
Viewer accounts read only — any POST/PUT/PATCH/DELETE is rejected for non-admins.
Least-privilege RBAC
The ServiceAccount gets namespaced read/patch in managed namespaces and read-only cluster-scoped access.
QUICKSTART
Up and running in three commands.
Build & push the images, create the namespace + auth secret, then apply the manifests. Replace the placeholders with your own values.
1
Build & push images
docker build -f deploy/Dockerfile.api \
  -t your-registry/ininfra-api:latest .
docker build -f deploy/Dockerfile.web \
  -t your-registry/ininfra-web:latest .

docker push your-registry/ininfra-api:latest
docker push your-registry/ininfra-web:latest
2
Namespace + auth secret
kubectl create namespace your-namespace

kubectl -n your-namespace create secret generic ininfra-auth \
  --from-literal=SESSION_SECRET="$(openssl rand -hex 32)" \
  --from-literal=ADMIN_USERNAME='admin' \
  --from-literal=ADMIN_PASSWORD='choose-a-strong-password'
3
Deploy
kubectl apply -f deploy/k8s/

kubectl -n your-namespace get pods
kubectl -n your-namespace logs deploy/ininfra-api
Configuration
Everything deployment-specific is an environment variable — nothing about a cluster, namespace, registry, or brand is hard-coded.
  • DATABASE_URLPostgres connection string
  • SESSION_SECRET>=32 random bytes
  • MANAGED_NAMESPACESCSV allowlist (default: default)
  • API_INTERNAL_URLin-cluster URL of the api
